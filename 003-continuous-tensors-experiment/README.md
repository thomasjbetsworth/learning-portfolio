# Continuous Tensors Experiment

Recently, I've been considering the similarity between discrete and continuous mathematical functions (sum vs. integral, finite differences vs. derivative, etc.). One thought which came to me was the idea of continuous tensors, or 'contensors'. In [this notebook](continuous-tensors.ipynb) I put this idea into code. Since there are (in theory) an uncountably infinite number of values in a contensor, there must be some procedure for finding them from their indices, rather than their being assigned arbitrarily. This is the 'rule' argument. The code isn't optimized for performance, but demonstrates the idea of contensors as I imagine them.

After researching the topic, I found that the equation for continuous matrix-vector multiplication is in essence identical to the [Fredholm integral equation](https://en.wikipedia.org/wiki/Fredholm_integral_equation), which I hadn't come across before. The only difference seems to be that contensors take the place of functions in the original equation. Through researching Fredholm's equation, I've realised that the procedural-generative nature of contensors effectively render them functions themselves, albeit with additional additive and multiplicative methods attached. For the most part, though, it seems I have merely coded a class which allows functions to be called with square brackets instead of standard ones. Ah, well, at least it agrees with the notation in [UDL](https://udlbook.github.io/udlbook/) now.
